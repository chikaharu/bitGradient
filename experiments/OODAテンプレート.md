# OODA テンプレート

実験/タスク 1 周ごとに本テンプレートを複製し、`OODA_history.json` に対応する 1 レコードを追記する。

---

## メタ

- cycle_id: (一意。例: `e165-c01`)
- task_ref: (Task#番号 または experiment ディレクトリ)
- 開始時刻: (ISO8601 / TZ=UTC)
- 担当: (agent/human/scheduler)

---

## O — Observe (観察)

直前の (結果 | 現象) を観察。

- 入力: (コーパス・コマンドライン・乱数種)
- 出力: (stdout 抜粋 / JSON / PNG / SVG パス)
- rustc verdict: (pass | fail | n/a)
- 異常/外れ値: (有無と短い記述)

## O — Orient (情勢判断)

高度な AI 判断で文脈解釈。

- 既知公理との整合: (A1〜A5 のどれと一致/反する)
- 反証/支持されたか: (対象命題と verdict)
- 先行実験との関係: (experiment-XXX の結果との比較)
- 想定外の発見: (有/無 + 1 行記述)

## D — Decide (意思決定)

選択可能な未来線を**実験計画法 (DOE)** で列挙する。

- 因子: (例: NSEG, k, 圧縮幅 d, アンカー方式)
- 水準: (各因子の取りうる離散値)
- 配列: (全数 / 直交配列 L4・L8・L9 / 部分要因)
- 評価指標: (rustc 通過率 / NDCG / σ_1²/Σ など)
- 候補未来線:
  - F1: (因子=水準 の組合せ + 期待効果)
  - F2: ...
  - F3: ...
- 採択: (F? を選ぶ理由)

## A — Act (実行)

Rust CLI バイナリとして実行。

- 実行コマンド: (`qsub "..."` または `cargo run --release -- ...`)
- 成果物パス: (実行ファイル / results/ / plot.png)
- 次サイクルへの観察対象: (次 Observe で見るべき値)

---

## 終了

- 終了時刻: (ISO8601)
- 結論: (1〜3 行)
- 次 cycle_id: (連結する場合)
